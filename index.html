Summary

Mutation explosion post 2006-ish

'Every man and his dog' starting making new databases, most of them for a new model.

Literally 100's of datastores to choose from since ~2015. A few competing to be the same as others just bigger, better and faster; Most a unique shape of their own.

Many developers started understanding how databases were composed - write-ahead logs, b-trees, lock-free algorithms, memory vs disk representations, logical clocks, etc. - and took those ingredients and composed new datastores.

Reduce your requirements, find more options on the shelf

  • Don't need disk persistence? Enjoy in-memory speed.
  • Dataset fits into one server? (Say 1TB RAM, 10TB disk) Distributed datastore not required.
  • Application-level data partitioning? Distributed datastore not required.
  • Don't need low response latency? HTTP API-using options OK.
  • Can re-initialize all data from external source? Datastores lacking backup method OK.
  • Don't need to share the data with other instances? Embedded storage engine or database becomes an option. Authentication unnecessary.
  • In a secure network? Can use datastores that lack TLS connection support.
  • Only need primary key, no secondary, compound, etc. indexes? Key-value db.
  • etc.

Many business requirements are strong restriction of options

  • Need data-at-rest encryption? Very few datastores.
  • Federated Authentication? Most enterprise datastores, few non-enterprise datastores.
  • Rapid restore of TB-sized backups? Filesystem snapshot methods are probably required.
  • Point-in-time restore? Not many datastores.

If it's in the DBA's hands they'll have to follow the rules above. All responsibility, no allowance for errors (except the sort billion-dollar vendors makes everyone suffer - you can't be fired for those.)

Only the application developer (or spokesperson for them eg. System architect) can absolve them from needing these. Eg. We'll do the encryption, use CSE; we'll split records into tables that are live and allow older ones to be frozen to reduce backup restore work.
